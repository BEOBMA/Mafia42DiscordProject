# 직업 배정 규칙 준수 점검 보고서

기준 규칙:

1. 고정 배정: **마피아 2명, 보조계열 1명, 의사 1명, 경찰 계열 1명**.
2. 나머지 슬롯: 위 4개 분류를 제외한 직업으로, **중복 없이** 구성.
3. 직업 선택 방식: 모든 플레이어 선호를 합산해 가중치를 만들고, 해당 가중치 기반 확률로 직업군을 선택.
4. 선택된 직업은 해당 직업을 선호한 플레이어들 중 랜덤으로 배정.
5. 단, **연인/비밀결사**는 선택되면 반드시 2명 배정.

## 코드 점검 결과

## 1) 고정 배정 규칙 위반 가능성
- `GameManager`는 고정 직업으로 마피아/의사/경찰만 강제하고, **보조계열 1명 고정 배정 로직이 없음**.
- 따라서 보조계열이 0명 또는 2명 이상이 될 수 있어 기준 규칙 위반 가능성이 있음.

관련 위치:
- `REQUIRED_MAFIA_COUNT`, `REQUIRED_DOCTOR_COUNT`, `REQUIRED_POLICE_COUNT`만 존재.
- 고정 배정 수행 함수도 보조계열 고정 수량을 다루지 않음.

## 2) 나머지 슬롯 직업군 제약 위반 가능성
- 비고정 직업 후보 선택 시 제외 대상이 `마피아/의사/경찰계열`만이며, **보조계열(Evil) 전체가 제외되지 않음**.
- 결과적으로 나머지 슬롯에서 보조계열이 또 선택될 수 있어 규칙(나머지는 보조계열 제외)에 어긋날 수 있음.

## 3) 중복 없는 직업 규칙 위반 가능성
- 비고정 직업 선택 자체는 중복 방지(`pickedNames`)를 적용.
- 하지만 마지막 잔여 인원 배정에서 `fallbackPool.randomOrNull()`로 재할당할 때 중복 방지가 없어,
  **최종 결과에서는 같은 직업이 여러 명에게 배정될 수 있음**.

## 4) 가중치 기반 선택의 일관성 문제
- 비고정 직업 선택은 전체 선호 빈도를 가중치로 사용해 확률 선택하므로 규칙과 대체로 부합.
- 다만, 선택 직업 배정 실패(선호 인원 부족) 시 이후 단계에서 랜덤 fallback으로 대체되어,
  **최종 배정이 선호/가중치 기반에서 이탈**할 수 있음.

## 5) 연인/비밀결사 2인 강제 규칙 위반 가능성
- 선택 단계에서 `연인/비밀결사`는 2슬롯 요구를 반영하지만,
  실제 배정 단계(`assignByPreference`)는 그 시점의 미배정 선호 인원이 2명 미만이면 배정을 건너뜀.
- 즉 선택됐더라도 실제 2명 배정이 실패할 수 있으며,
  이후 fallback이 임의 배정으로 덮어써 **"항상 2명" 보장이 깨짐**.

## 보완 적용 내역 (2026-03-18)
아래 항목은 이후 코드 보완에서 실제 반영됨.

1. 고정 배정에 `보조계열 1명`을 추가.
2. 비고정 후보를 `비-Evil` + `마피아/의사/경찰계열 제외`로 제한.
3. 비고정 선택 단계에서 즉시 배정하고, 당시 `미배정 선호 인원` 수를 기준으로 배정 가능성을 검증.
4. 경찰계열/보조계열 고정 직업 선택은 모두 선호 가중치 기반 확률로 선택.

## 잔여 리스크
- 선호 기반 비고정 배정이 슬롯 수를 끝까지 채우지 못하는 극단 케이스에서는
  보완 단계가 랜덤 요소를 포함하며, 이 경우 이론적으로 규칙 완전 일치가 깨질 수 있음.
- 위 케이스를 완전히 제거하려면, 전체 배정을 만족할 때까지 재추첨/백트래킹하는
  제약 충족(Constraint Satisfaction) 방식이 추가로 필요함.
